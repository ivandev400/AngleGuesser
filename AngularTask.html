1. Create an Initial Screen with:

  • Player Registration Form
  • Scores Table

1.1 Player Registration Form

  • Use a Reactive Form (FormGroup) with 3 fields:
    • First Name
    • Last Name
    • Nick Name
  • First and Last Name fields must be combined inside <app-name-fields> component.
  • Validation for minLen and maxLen (passed as component input).
  • Clear button resets form and messages.
  • Start Game button:
    • Validates fields.
    • Checks for duplicate nicknames in score table (service).
    • On success, navigates to Game Screen.
    • On error, shows appropriate error message.

Use this HTML structure:

<form [formGroup]="fg" (ngSubmit)="StartGame()" class="main-form">
  <label class="form-label">
    First and Last Name:
    <app-name-fields [minLen]="10" [maxLen]="10" formControlName="nameFields"></app-name-fields>
  </label>
  <br />
  
  <label class="form-label">
    Nick Name:
    <input formControlName="nickName" class="form-input" />
  </label>
  <br />

  <app-message [error]="successMessage" severity="success"></app-message>
  <app-message [error]="errorMessage" severity="error"></app-message>

  <button type="button" (click)="clear()" class="btn btn-clear">Clear</button>
  <button type="submit" class="btn btn-save">Start the game</button>
</form>

<app-scores-table></app-scores-table>

1.2 Scores Table

  • Component: <app-scores-table>
  • Displays player records in a simple table with columns:
    • Full Name
    • Nick Name
    • Level
    • Average Accuracy

1.3 Score Management

  • Store player info (full name, nickname, level, average accuracy) in a service.
  • Data resets after browser refresh (not persisted).

1.4 Angular Signals

  • Use Angular Signals to store scores and trigger updates in the Scores Table.
